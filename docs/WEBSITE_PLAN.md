# LifeOS Website Plan

The LifeOS website is the public showroom for the project.

It should explain the product, show screenshots, document releases, and present TimerAgent as the first shipped desktop-only utility.

The website is not the LifeOS app.

## Website Purpose

The first website should help someone understand:

- What LifeOS is
- Why weekly pressure matters
- How desktop and mobile fit together
- What TimerAgent does
- What has already shipped
- What is planned next
- Where the GitHub repo is
- What the roadmap looks like

## Website Positioning

LifeOS is a weekly pressure command centre.

It connects money, work, deductions, appointments, tasks, messages, and follow-ups into one weekly decision layer.

The main question is:

```text
What is putting pressure on this week, and what is safe to do next?
```

## Home Page Sections

Possible home page structure:

1. Hero
   - LifeOS: weekly pressure command centre.
   - Desktop power version. Mobile optimized version. Shared core.

2. Problem
   - People already juggle bank apps, calendars, notes, pay-later apps, reminders, screenshots, and memory.

3. Product
   - LifeOS turns that pressure into a clear weekly dashboard.

4. First shipped module
   - TimerAgent tracks work, billable time, earned income, tax set-aside, and CSV logs.

5. Platform model
   - Desktop leads.
   - Core is shared.
   - Mobile pressure-tests the model.

6. Roadmap
   - Money pressure
   - Pay-later
   - Agenda
   - Follow-ups
   - Weekly close-out
   - Integrations later

7. GitHub links

## What Not To Build Yet

- Login portal
- Cloud dashboard
- Payment system
- Public SaaS accounts
- Mobile sync backend
- Client portal
- Bank integration
- Automated SMS system

## First Website Tech Direction

Keep it simple.

Good first options:

- Static HTML/CSS
- GitHub Pages
- Vite/React static site
- Astro static site
- Simple docs-style site

Do not build a backend until LifeOS itself needs one.

## v0.1 Website Goal

Plan only.

The first website can be built later as a static product/portfolio page.
