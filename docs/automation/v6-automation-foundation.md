# v6 automation foundation

LifeOS automation begins as visible, explainable and reversible intent. A reviewed source snapshot is evaluated against a versioned rule using deterministic conditions. The result preserves condition pass/fail details, source trust state, target, risk, approval policy, execution mode and a stable duplicate key.

A matching safe rule may create a proposal. A forbidden high-risk rule creates blocked proof instead. Approval or rejection is audited and retained; neither path mutates Follow-Ups, Work Pipeline, Evidence, Money, Gmail, Calendar or any external system.
