# LifeOS v1.6 Release Notes

## Release

**Version:** v1.6  
**Screenshot group:** Group 02 — Daily Operating Flow  
**Build lane:** Local-first WPF desktop foundation  
**Status:** Screenshot group captured after the Daily Operating Flow workspace was visible in the app.

## What v1.6 adds

v1.6 adds the Daily Operating Flow layer.

The goal is to control the day without turning every thought into a task. Daily Operating Flow gives LifeOS a practical day-control workspace built around:

- visible daily blocks
- anchors
- next actions
- waiting checkpoints
- low-energy fallback blocks
- proof capture reminders
- recovery blocks
- stop points

This is not a calendar replacement and not a full scheduling engine. It is a local-first operating layer that helps decide what can move today.

## Design rule

v1.6 keeps the existing page model. It does not attempt the later major UI/workspace reshape.

The later UI reshape remains intentionally deferred until after integrations and the AI assistant layer prove the real workflows.
