# LifeOS v1.7 Release Notes

## Release

**Version:** v1.7  
**Screenshot group:** Group 03 — Paid Work / Money / Proof  
**Build lane:** Local-first WPF desktop foundation  
**Status:** Screenshot group captured after the Paid Work / Money / Proof workspace was visible in the app.

## What v1.7 adds

v1.7 strengthens the paid-work spine of LifeOS.

The goal is to turn work into controlled, proof-backed money without pretending expected money is safe money. It ties work sessions, pipeline value, invoice readiness, proof readiness, pending manual income, and copy-ready admin updates into one conservative local workspace.

v1.7 helps answer:

- What work can move today?
- What needs timesheet evidence?
- What is invoice-ready or invoice-review ready?
- What expected money exists but is not safe yet?
- What manual income is pending?
- What proof exists or still needs attaching?
- What client/admin update can be reviewed and sent manually?

## Group 03 scope

Group 03 covers:

1. **Command Centre**
   - shows the app still opens cleanly after v1.7
   - displays v1.7 command-centre state and existing pressure signals

2. **Paid Work / Money / Proof — top**
   - shows the main v1.7 workspace
   - captures health, admin actions, invoice-ready value, unpaid billable value, expected pipeline, pending manual income, and proof readiness

3. **Money/proof gate + focus queue**
   - shows the conservative money/proof gate
   - keeps expected pipeline and pending manual income separate from safe money
   - surfaces paid-work focus queue items

4. **Copy-ready client/admin update**
   - shows the manual-only update text
   - proves LifeOS can prepare a client/admin summary without sending anything automatically

5. **Legacy invoice-ready boundary**
   - keeps the older invoice-ready summary visible for continuity
   - states the v1.7 boundary clearly

6. **Proof Tracker context**
   - shows the older Proof Tracker module remains available
   - keeps proof capture separate while v1.7 provides the stronger paid-work gate

## Boundary

v1.7 is **not** accounting automation.

It does not:

- send invoices
- collect payments
- calculate tax
- connect to accounting systems
- decide expected money is safe
- send emails/messages automatically

It stays local-first, manual-review, proof-aware, and conservative.
