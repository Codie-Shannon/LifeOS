# LifeOS v1.8 Release Notes

## Release

**Version:** v1.8  
**Screenshot group:** Group 04 — Settings / Safety / Theme  
**Build lane:** Local-first WPF desktop foundation  
**Status:** Screenshot group captured after Settings / Safety / Theme was visible in the app and the sidebar navigation scroll hotfix made lower modules reachable.

## What v1.8 adds

v1.8 adds the Settings / Safety / Theme foundation.

The goal is to protect the offline foundation before integrations. This version gives LifeOS an explicit safety and preference layer for:

- local-only mode
- manual review gates
- expected-money safety
- screenshot privacy
- destructive-action confirmations
- demo-safe data
- theme preference foundation
- accent preference foundation
- build-lane/version notes

## Theme boundary

v1.8 includes a **theme preference foundation**, not a full live theme engine.

It does:

- store appearance preference
- store accent preference
- show current theme state
- keep the current dark/cyan visual lane consistent
- document the safety boundary around screenshots and demo privacy

It does not yet:

- recolour every control dynamically when accent changes
- provide a full theme editor
- support custom palettes
- apply multiple complete visual themes across every module

The correct wording is **theme foundation** or **theme preferences**, not full dynamic theming.

## Group 04 scope

Group 04 covers:

1. **Command Centre**
   - shows the app still opens cleanly after v1.8
   - displays v1.8 command-centre state and existing pressure signals

2. **Settings / Safety / Theme top**
   - shows safety mode, guardrail counts, manual gates, privacy protections, appearance, accent, local-first state, and expected-money safety
   - shows the sidebar scroll hotfix working because Settings / Safety is reachable

3. **Settings / Safety / Theme controls + guardrails**
   - shows safety controls, appearance/accent preferences, local-only mode, manual review, expected-money protection, screenshot privacy, destructive-action confirmation, and demo-safe data controls

## Navigation hotfix

v1.8 added enough navigation entries that lower modules were cut off at some window heights. The v1.8.1 hotfix wraps sidebar navigation in a scroll area so Settings / Safety, TimerAgent, Projects, and Relationship Radar remain reachable.

## Boundary

v1.8 is **not** integrations, cloud sync, AI assistant execution, or automatic decision-making.

It protects the local desktop foundation before those layers arrive.

## Privacy rule

- real company names
