# LifeOS v13 Group 65 - Household Inventory, Meals, Stores and Price Context

Group 65 extends the v13 Household and Grocery lane beyond shopping execution into household inventory planning.

## Added

- Household inventory records with explicit stock states: out, low, enough, overstocked and unknown.
- Inventory review candidates for out, low, unknown and expiring stock.
- Store profiles for local planning context without treating store data as a trusted price feed.
- Meal recipe and ingredient foundations that can identify ingredient gaps.
- Desktop Household/Grocery views for inventory, meal ingredients and store profiles.
- Full Mobile inventory review surface for stock state and ingredient-gap review.
- Core regression coverage for stock-state counts, review-only grocery candidates, overstock protection, recipe gaps and store-profile trust boundaries.

## Boundaries

- Inventory does not automatically create or mutate grocery lists.
- Meal ingredient gaps do not clone into shopping lists without review.
- Store profiles are planning context only, not live feeds or trusted prices.
- No automatic ordering, payment initiation, external-cart mutation or provider write is active.
- All proof data is synthetic.
