# LifeOS v2.0 Release Notes

## Release

**Version:** v2.0  
**Screenshot group:** Group 05 — v2 Paid Desktop Release  
**Build lane:** Local-first paid desktop release candidate  
**Status:** Screenshot group captured after the Desktop Release Centre was visible in the app.

## What v2.0 adds

v2.0 turns the offline LifeOS foundation into a paid desktop release candidate checkpoint.

The goal is not to ship integrations, cloud sync, accounting automation, automatic messaging, or AI execution yet. The goal is to prove that the local desktop app has enough operating structure, proof safety, release discipline, documentation, and screenshots to be treated as a serious paid desktop foundation.

v2.0 adds:

- Desktop Release Centre
- paid desktop release readiness profile
- local JSON release-readiness storage
- release readiness scoring
- release review checklist
- completed foundation checklist
- future/planned release items
- Command Centre v2.0 state
- explicit v2.0 boundary
- release docs and screenshot group

## Important version note

Some module pages may still show their module version, such as **Settings / Safety / Theme • v1.8**.

That is intentional.

v2.0 is the overall desktop release checkpoint. Existing module labels can remain at the version where that module was introduced or last materially changed. Settings / Safety was introduced in v1.8 and is carried forward as part of the v2.0 paid desktop release foundation.

## Group 05 scope

Group 05 covers:

1. **Command Centre**
   - shows the app opens cleanly at v2.0
   - confirms the overall shell is still intact
   - shows v2.0 command-centre state

2. **Desktop Release Centre top**
   - shows release state, readiness score, completed checks, review checks, blocked checks, planned next, local-only safety, and screenshot-safe status

3. **Release readiness review checks**
   - shows release readiness reasons
   - shows release controls
   - shows review items required before/for v2.0 final docs

4. **Boundary and local release file**
   - shows planned post-v2.0 items
   - shows v2.0 boundary
   - shows the local release-readiness file path

## v2.0 boundary

v2.0 is the paid desktop release readiness checkpoint for the offline/local-first app.

It is not:

- integrations
- cloud sync
- automatic messaging
- accounting automation
- AI assistant execution
- mobile capture
- production installer/signing system
- full dynamic theme engine

- real company names
