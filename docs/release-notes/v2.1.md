# LifeOS v2.1 Release Notes

## Release

**Version:** v2.1  
**Screenshot group:** Group 06 — Universal Spine  
**Build lane:** Local-first offline OS foundation  
**Status:** Screenshot group captured after the Universal Spine workspace was visible in the app.

## What v2.1 adds

v2.1 adds the Universal Spine foundation.

The Universal Spine is the local cross-module link layer that lets LifeOS connect context without pretending integrations, universal search, inbox scanning, cloud sync, or AI assistant reasoning are active yet.

It gives LifeOS a local structure for connecting:

- work
- money
- proof
- relationships
- daily flow
- evidence
- release state
- settings/safety
- knowledge/docs

## Why this matters

Before integrations and AI assistant layers, LifeOS needs a stable internal spine for linking module state.

v2.1 proves that the app can represent cross-module relationships such as:

- paid work needing proof
- work producing expected money
- relationship follow-ups affecting paid work
- daily operating blocks supporting paid work
- screenshot privacy protecting docs/proof
- release state depending on screenshot docs
- docs/version history staying current

## v2.1 boundary

v2.1 is the Universal Spine foundation.

It is not:

- universal search
- AI reasoning
- inbox scanning
- integrations
- cloud sync
- automatic decisions
- automatic messaging
- final workspace redesign

Those layers come later after the offline spine is stable.
