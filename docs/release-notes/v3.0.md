# LifeOS v3.0 Release Notes

## Release

**Version:** v3.0  
**Screenshot group:** Group 07 — OS Navigation / Core Modules  
**Build lane:** Local-first offline OS foundation  
**Status:** Screenshot group captured after the OS Navigation Centre was visible in the app.

## What v3.0 adds

v3.0 adds the OS Navigation / Core Modules foundation.

The goal is to strengthen the offline app shell without doing the later full workspace redesign yet. v3.0 keeps the existing WPF page model and adds a local operating map for the growing module list.

v3.0 adds:

- OS Navigation Centre
- visible/core/pinned module map
- workspace group map
- modules needing review
- planned/future module separation
- sidebar reachability proof
- v3.0 boundary notes
- local OS navigation storage
- Command Centre v3.0 state

## Why this matters

LifeOS is becoming a real offline OS-style app, not just a pile of pages.

Before integrations, universal search, advanced knowledge, or AI assistant layers are added, the app needs a stable module/navigation structure so future integration signals have obvious places to land.

## Group 07 scope

Group 07 covers:

1. **Command Centre**
   - shows the app opens cleanly at v3.0
   - confirms the shell remains intact
   - shows v3.0 command-centre state

2. **OS Navigation Centre top**
   - shows visible modules, core modules, pinned modules, workspace groups, review-needed modules, planned/future modules, hidden modules, sidebar scroll, UI reshape state, and local-first sync

3. **OS navigation rule and controls**
   - shows the rule explaining sidebar scrolling, workspace groups, protected core modules, deferred UI reshape, and local-first state
   - shows reset/demo navigation controls

4. **Pinned/core operating modules**
   - shows protected/pinned operating modules like Command Centre, Money Pressure, Work Sessions, Paid Work Centre, and other core workflow areas

5. **Workspace group map**
   - shows grouped workspaces such as command, money/paid work, proof/evidence, daily operating flow, relationships/follow-ups, system spine/safety, and planned knowledge layer

6. **Modules needing v3 review and planned/future modules**
   - shows Projects and TimerAgent needing review
   - shows Advanced Search and Knowledge Base as planned/future modules, not complete work

7. **Boundary and local navigation file**
   - confirms v3.0 is not the major workspace redesign, universal search, advanced knowledge layer, integrations, cloud sync, AI assistant execution, or automatic decision-making
   - shows local OS navigation storage path

## v3.0 boundary

v3.0 strengthens OS navigation and core modules.

It is not:

- the major workspace redesign
- universal search
- advanced knowledge layer
- integrations
- cloud sync
- AI assistant execution
- automatic decision-making
- full dynamic theming

Those layers come later after the offline OS foundation is stable.
