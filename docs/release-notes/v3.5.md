# LifeOS v3.5 Release Notes

## Release

**Version:** v3.5  
**Screenshot group:** Group 08 — Advanced Modules / Search / Knowledge  
**Build lane:** Local-first offline OS foundation  
**Status:** Screenshot group captured after the Search / Knowledge Centre was visible in the app.

## What v3.5 adds

v3.5 adds the local Advanced Search / Knowledge foundation.

The goal is not to turn on external search, full-text indexing, connector search, inbox scanning, cloud sync, integrations, or AI reasoning yet. The goal is to create the local shape that those later layers can safely feed.

v3.5 adds:

- Search / Knowledge Centre
- local knowledge item model
- local search profile model
- knowledge source boundaries
- manual review state
- planned/future knowledge items
- local search profile scoping
- Command Centre v3.5 state
- explicit v3.5 boundary
- local JSON knowledge storage
- demo-safe knowledge/search data

## Why this matters

Before integrations and AI assistant workflows, LifeOS needs a clear local knowledge layer.

Future integrations should not dump data into chaos. They should feed defined places:

- release notes and screenshot history
- money and proof safety
- relationship and follow-up state
- daily operating control
- future integration landing zones
- Universal Spine and OS Navigation context

v3.5 proves the local structure without pretending the external layers are active.

## v3.5 boundary

v3.5 is local Search / Knowledge structure only.

It is not:

- external search
- full-text indexing
- connector search
- inbox scanning
- cloud sync
- AI reasoning
- automatic summarisation
- final knowledge automation
- integrations
- major workspace redesign

Those layers come later after the offline foundation is stable.
