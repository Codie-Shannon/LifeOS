## What v4.1 adds

v4.1 turns the v4.0 LifeOS Spine Map into a working item/state engine.

It adds:

- stateful item model
- source kind model
- impact areas
- risk levels
- item transition rules
- review queue
- pressure item list
- money-impact items
- work/people/proof items
- local item-state storage
- Item State Engine desktop workspace
- Command Centre v4.1 item-state signals

## Boundary

v4.1 models local item state and transition rules.

It does not connect real APIs, request OAuth permissions, sync live email/calendar/accounting/bank data, run AI actions, build the companion app, or perform the major workspace redesign.
