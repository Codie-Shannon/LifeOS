# LifeOS v4.1 Release Notes

## Release

**Version:** v4.1

## What v4.1 adds

v4.1 turns the v4.0 LifeOS Spine Map into a working local item/state engine.

It adds:

- stateful item model
- source kind model
- item impact areas
- risk levels
- item transition rules
- review queue
- command-centre pressure item list
- money-impact item list
- work/people/proof item list
- local item-state JSON storage
- Item State Engine desktop workspace
- Command Centre v4.1 item-state signals

## Master rule

```text
Everything important becomes an item.
Every item has state.
Every state affects pressure.
Every pressure feeds the Command Centre.
```

## Review-first rule

External/OCR/uncertain data must not become trusted state automatically.

The correct v4.1 flow is:

```text
Source / OCR / future integration / manual input
-> stateful item
-> review queue if untrusted
-> evidence/source check
-> transition rule
-> trusted state only after review
-> pressure signal
-> Command Centre
```

## Command Centre signal

Command Centre now shows v4.1 item-state signals including:

- State items
- Needs review
- Untrusted

It also includes an Item State Engine panel explaining why Command Centre receives pressure from item state rather than random raw records.

## State transition rules

v4.1 maps guarded state transitions such as:

- Bill approaching
- Mark bill paid
- Mark BNPL paid
- Confirm OCR receipt
- Payment expected
- Mark invoice paid
- Unblock work
- Do not chase
- Follow-up due
- Accept import
- Reject import
- Close week item

Transition rules can require evidence, manual review, or destructive-action caution.

## Boundary

v4.1 does not:

- connect real APIs
- request OAuth permissions
- sync live email/calendar/accounting/bank data
- run AI actions
- automatically change real-world systems
- build the companion app
- perform the major workspace redesign
