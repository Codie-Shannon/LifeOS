# LifeOS v4.2 Release Notes

## Release

**Version:** v4.2  
**Screenshot group:** Group 12 - Bills / Upcoming Payments / Pay Later  
**Build lane:** LifeOS spine completion  
**Status:** v4.2 code and Group 12 screenshots prove the local money-obligation engine.

## What v4.2 adds

v4.2 adds the Bills / Payments engine.

It models:

- bills
- subscriptions
- upcoming payments
- Pay Later / Zip / Afterpay instalments
- Pay Later remaining balances
- hidden deductions
- tax/GST/ACC/student-loan/KiwiSaver/custom reserve placeholders
- manual cashflow items
- review/evidence gates
- safe-to-spend drag
- due pressure
- local JSON storage
- Command Centre v4.2 money-obligation signals

## Money rule

```text
Expected money is not safe money.
Bills and Pay Later reduce safe-to-spend until paid/cleared with evidence.
Hidden deductions must be visible before spending.
Paid state requires evidence or a trusted source note.
```

## Command Centre signal

Command Centre now shows v4.2 money-obligation signals including:

- Money obligations
- Due today
- Pay later load
- Hidden deductions
- Safe drag

It also includes a Bills / Upcoming Payments / Pay Later panel explaining why money pressure comes from stateful obligations rather than raw guesses.

## Why this matters

v4.2 preserves the original LifeOS money logic:

- bills must be visible before they become urgent
- BNPL must be tracked as real pressure
- remaining Pay Later balance matters
- hidden deductions lower confidence in safe-to-spend
- payment is not trusted until evidence exists
- OCR/imported money data must be reviewed before it affects trusted state

## Boundary

v4.2 does not:

- connect bank feeds
- connect Pay Later providers
- connect accounting systems
- sync live email/calendar/accounting/bank data
- run OCR automation
- request OAuth permissions
- run AI actions
- automatically change real-world systems
- build the companion app
- perform the major workspace redesign
