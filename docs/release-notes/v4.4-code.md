# LifeOS v4.4 Code Checkpoint

## Release lane

**v4.4 - Agenda + Payment Calendar**

## Summary

v4.4 adds a local/manual Payment Calendar workspace.

This turns dates into a first-class operating lane before real integrations:

- bills due by date
- subscriptions due by date
- Pay Later / Zip / Afterpay instalments
- expected money dates
- agenda/family/work commitments
- follow-up windows
- receipt/import review windows
- weekly close-out timing
- payment evidence gates
- calendar sync boundaries

## Why this matters

LifeOS already models money pressure, stateful items, bills/payments, and safe-to-spend.

v4.4 adds the missing date lane:

> Dates create visibility, not trust.

A bill being due, expected money arriving, or calendar item existing does not automatically change trusted LifeOS state. Review/evidence gates still apply.

## Added

- `src/LifeOS.Core/PaymentCalendar`
- `LifeOS.Shared/PaymentCalendar`
- `Payment Calendar` navigation button
- Payment Calendar workspace
- Command Centre v4.4 calendar cards/signals
- Agenda page v4.4 connection wording
- Local JSON storage: `lifeos-payment-calendar.json`

## Boundary

v4.4 is local/manual only.

It does not connect:

- Google Calendar
- Outlook Calendar
- Gmail/Outlook email
- bank feeds
- Pay Later/BNPL providers
- accounting systems
- open banking
- OCR automation
- OAuth
- AI actions
- companion app
- major workspace redesign
