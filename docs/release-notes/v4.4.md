# LifeOS v4.4 Release Notes — Agenda + Payment Calendar

## Summary

LifeOS Desktop v4.4 adds the Agenda + Payment Calendar lane before v5 integrations. Bills, subscriptions, BNPL instalments, expected-money dates, agenda commitments, review windows, and fixed commitments now sit in one time-aware view.

## What changed

- Added the Payment Calendar module.
- Added the `Payment Calendar` navigation entry.
- Added date-aware agenda/payment models and local JSON storage.
- Added demo-safe date items for:
  - bills,
  - subscriptions,
  - Pay Later / Zip / Afterpay instalments,
  - expected income dates,
  - receipt/import review windows,
  - agenda/family commitments,
  - vehicle/compliance renewal style payments.
- Added top-level Payment Calendar metrics:
  - date items,
  - this week,
  - today,
  - due soon,
  - overdue,
  - payment dates,
  - agenda dates,
  - fixed commitments,
  - pay-later dates,
  - expected income.
- Added time-aware lane grouping by date.
- Added separate sections for:
  - Today / next two days,
  - Payment dates / safe-to-spend lane,
  - Agenda commitments / time lane,
  - Pay Later / Zip / Afterpay schedule,
  - Expected money dates excluded from safe money,
  - Review windows / untrusted date items.
- Updated Command Centre metrics to expose v4.4 calendar lane signals.
- Kept external integrations deliberately off.

## Operating rule

Dates do not make money safe. Bills, BNPL instalments, agenda commitments, payment dates, expected income, and review windows must be visible together before the week is trusted.

## Safety rule

Due dates do not mark bills paid. Payment evidence is still required before money items become trusted.

## Boundary

v4.4 models agenda/payment dates locally. It does not connect Google Calendar, Outlook Calendar, Gmail/Outlook email, bank feeds, BNPL providers, accounting systems, open banking, OCR automation, OAuth, or AI actions.
