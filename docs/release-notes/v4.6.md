# LifeOS Desktop v4.6 - Receipt OCR / Evidence-to-Item

## Summary

v4.6 adds a local, review-first Receipt OCR / Evidence-to-Item operating lane. Receipt or OCR fields are treated as candidates rather than trusted facts until a human reviews the source and accepts the item.

## Added

- Receipt evidence candidate model and calculator.
- Local receipt evidence storage.
- Receipt OCR / Evidence navigation and workspace.
- Review queue for untrusted candidates.
- Missing-source evidence lane.
- Accepted/trusted-after-review lane.
- Money-impact candidates.
- Paid-work/proof candidates.
- Command Centre receipt evidence signals.
- Local demo reset control.
- Clear v4.6 boundary and v4.7 next-lane marker.

## Trust rule

A receipt candidate does not affect trusted item state or money calculations until it has source evidence and an accepted state.

## Boundary

v4.6 remains local/manual. It does not run live OCR, scan images, import email attachments, connect bank/accounting systems, create invoices, create payment state, or run AI actions.
