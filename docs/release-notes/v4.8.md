# LifeOS v4.8 - Command Centre Pressure Engine

## Summary

v4.8 turns reviewed local module output into one ranked pressure engine.

Instead of treating every open record as equal, LifeOS now separates pressure into:

- Act now
- Review before action
- Waiting / do not chase
- Protected / suppressed

## Added

- Core pressure severity and lane models.
- Ranked local pressure signals.
- Configurable local severity weights and pressure thresholds.
- Top-signal limit.
- Conservative suppression rules.
- Untrusted-review gate.
- Command Centre pressure score and lane metrics.
- Top-ranked pressure signal cards.
- Next-safest-action output.
- Module pressure bridge across:
  - Money Profile and Bills / Payments
  - Payment Calendar
  - Work Pipeline
  - Receipt OCR / Evidence
  - Weekly Close-Out
  - Daily Operating Flow
- Local pressure-policy storage.
- v4.8 boundary and v4.9 readiness section.

## Safety behaviour

- Critical and due-now items rise first.
- Untrusted imports and incomplete evidence require review.
- Waiting-on-others and blocked work remain visible without hijacking active work.
- Expected money remains excluded from safe money.
- Protected signals remain contained.
- The engine ranks source-module signals; it does not replace the source modules.
- No automatic external actions are performed.

## Boundary

v4.8 is local ranking and suppression only. It does not send messages, pay bills, move money, close projects, create invoices, accept OCR automatically, change external calendars, sync external systems, or run AI actions.
