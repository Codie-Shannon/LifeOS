# LifeOS Desktop v4.9 - Integration Inbox + v5 Readiness

## Summary

v4.9 completes the v4 operating spine by adding a review-first Integration Inbox and connector-readiness contracts for future v5 integrations.

## Added

- Read-only integration preview records.
- Source type, source label, external reference, duplicate key, source evidence, and review-note provenance.
- Review states for new, needs review, accepted, deferred, rejected, duplicate suspected, and linked previews.
- Explicit Accept, Defer, Reject, and Link demo controls.
- Duplicate-suspicion blocking.
- Manual target-module handoff contracts.
- Source lanes for email, calendar, accounting, files, OCR, and future banking previews.
- v5 connector-readiness matrix.
- Command Centre integration preview, review, conflict, and imported-value signals.
- Local Integration Inbox storage.

## Safety rules

- Raw connector records never become trusted LifeOS items automatically.
- Imported money is not safe money.
- Accepted previews do not mutate target modules automatically.
- Provenance and duplicate state remain visible.
- Target modules remain authoritative and validate their own rules.
- Live connectors, OAuth, polling, and AI actions remain disabled.
