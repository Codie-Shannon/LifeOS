# v5.0-alpha — Group 22 product checkpoint

Group 22 introduces the first narrow authenticated provider boundary: Google Calendar read-only.

The connector requests only `calendar.readonly`, uses explicit local connection and manual refresh actions, enforces a bounded date range, and maps provider events into the existing Integration Inbox as untrusted read-only previews. Duplicate suspicion, provenance, audit history, human acceptance, and separate target-module linking remain intact.

This checkpoint does not add calendar writes, background synchronization, scheduled refresh, email access, automatic preview acceptance, automatic target-module updates, or Group 23 behavior.

Provider-console configuration and real-account verification remain local setup steps and must not be falsely represented as completed by source-only validation.
