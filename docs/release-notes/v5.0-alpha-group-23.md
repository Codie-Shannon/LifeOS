# v5.0-alpha — Group 23 connector lifecycle hardening

Group 23 hardens the existing Google Calendar read-only connector without adding automation or another provider.

Implemented:
- normalized lifecycle states and persisted non-secret status metadata
- configuration and local token-cache visibility
- last refresh attempt, last successful refresh, bounded range and result counts
- sanitized connector error persistence
- explicit reconnect, retry, disconnect and local-cache-clear actions
- empty, partial, failed, expired/revoked and missing-token states
- connector lifecycle audit entries in the existing integration audit trail
- imported previews remain after disconnect or local cache clearing

Still excluded: scheduled refresh, background polling, automatic retry/reconnect, calendar writes, Gmail, Outlook, Email Radar and Group 24 work.
