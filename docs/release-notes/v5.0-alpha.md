# LifeOS Desktop v5.0-alpha - Connector Foundation and First Live Read-Only Connector

## Summary

v5.0-alpha now includes guarded local CSV, JSON, and ICS preview imports plus one narrow authenticated Google Calendar read-only connector.

Google Calendar events are retrieved only after explicit connection and a manually confirmed bounded date range. They enter the existing Integration Inbox as untrusted previews with provenance, duplicate protection, audit history, and mandatory human review. Neither Google Calendar nor operational LifeOS modules are changed automatically.

## Active local connector behavior

- Local CSV preview import.
- Local JSON preview import.
- Local ICS calendar preview import.
- Explicit confirmation before local imported previews are saved.
- Duplicate detection with visible duplicate suspicion.
- Preserved source provenance and import audit history.
- Human Accept, Defer, Reject, and Link review controls.
- Reviewed connector-preview signals in the Command Centre pressure engine.

## Active Google Calendar behavior

- One Google Calendar provider only.
- OAuth connection using a local desktop configuration boundary.
- `calendar.readonly` permission only.
- Explicit user connection confirmation.
- Manual refresh only.
- User-confirmed bounded date ranges with a 31-day maximum.
- Event mapping through the existing `IntegrationPreviewIntake` boundary.
- Stable external references and duplicate keys.
- Provider/calendar provenance and source evidence.
- Duplicate-suspected repeated refresh behavior.
- Explicit disconnect and local token-cache deletion.

## Verified live proof

Group 22 verified the provider against a real Google test account using safe sample events:

- connection succeeded with read-only scope
- five events were retrieved during the first bounded refresh
- zero malformed events were reported
- events appeared as untrusted previews requiring human review
- repeating the same range produced five duplicate-suspected previews
- disconnect deleted the local token cache
- existing preview evidence remained available after disconnect

## Safety rules

- External data is untrusted by default.
- Imported records remain read-only previews.
- Confirmation is required before retrieval/import.
- Duplicate suspicion remains visible and blocks unsafe acceptance.
- Audit history and provenance remain preserved.
- Accepted previews do not mutate target modules automatically.
- Target modules remain authoritative and must validate any later manual handoff.
- Imported or expected money remains excluded from safe money.
- No client secret, access token, refresh token, or private account identifier is committed.

## Inactive capabilities

- No calendar write access.
- No Gmail, Outlook, or inbox scanning.
- No bank feed or money movement.
- No scheduled refresh, startup refresh, background polling, or silent synchronization.
- No automatic external mutation.
- No automatic LifeOS target-module mutation.
- No AI action or autonomous connector behavior.
