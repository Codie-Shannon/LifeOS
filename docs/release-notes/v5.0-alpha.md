# LifeOS Desktop v5.0-alpha - Local Connector Foundation

## Summary

v5.0-alpha aligns the LifeOS product state with the implemented local connector foundation. Local CSV, JSON, and ICS files can be imported into the Integration Inbox as guarded, read-only previews.

## Active connector behavior

- Local CSV preview import.
- Local JSON preview import.
- Local ICS calendar preview import.
- Explicit confirmation before imported previews are saved.
- Duplicate detection with visible duplicate suspicion.
- Preserved source provenance and import audit history.
- Human Accept, Defer, Reject, and Link review controls.
- Reviewed connector-preview signals in the Command Centre pressure engine.

## Safety rules

- External data is untrusted by default.
- Imported records remain read-only previews.
- Confirmation is required before import.
- Duplicate suspicion remains visible and blocks unsafe acceptance.
- Audit history remains preserved.
- Accepted previews do not mutate target modules automatically.
- Target modules remain authoritative and must validate any later manual handoff.
- Imported or expected money remains excluded from safe money.

## Inactive capabilities

- No Gmail, Outlook, Microsoft 365, or Google Calendar authentication.
- No live OAuth flow or API synchronization.
- No inbox scanning or polling.
- No bank feed or money movement.
- No automatic external mutation.
- No automatic LifeOS target-module mutation.
- No AI action or autonomous connector behavior.
