# LifeOS v6.0.0-alpha.2 — Group 28

## Guarded internal automation

Group 28 adds the first deliberately narrow execution bridge.

Approval remains separate from execution. A proposal must first be explicitly approved, remain current and eligible, pass immediate policy revalidation, display a final exact before/after preview, and receive a separate explicit final confirmation.

## Executable allowlist

Group 28 includes one typed executable action:

- add one internal review note to a fictional/internal target

The action is Low risk and reversible. It cannot target an external system, financial state, communication channel, mailbox, calendar, destructive operation, script, process, plugin or AI decision.

Medium Follow-Up draft proposals remain proposal-only.

## Execution and Undo

A successful execution retains:

- proposal and rule references
- exact target
- source snapshot
- approved and final-confirmed timestamps
- before snapshot
- after snapshot
- success state
- reversibility and Undo availability
- audit references

Undo requires explicit user action. It restores the exact prior fictional internal state only when the target has not changed incompatibly. Undo cannot run twice and does not delete execution history or audit.

## Stale-state safety

Eligibility is recalculated immediately before execution.

The screenshot proof changes the fictional source after approval. LifeOS marks the proposal `Stale`, blocks execution, records the source-state mismatch and requires a new dry run. The original approval remains retained.

## Global gate and blocked capabilities

Guarded execution starts paused. Dry-run and approval remain available while final execution is blocked.

The following remain blocked:

- external writes
- communication send
- mailbox mutation
- calendar mutation
- financial mutation
- destructive action
- scripts and arbitrary processes
- plugins
- AI-generated rules or decisions

There is no background worker, scheduler, timer-triggered execution, startup execution, unattended execution, timeout execution or automatic retry.
