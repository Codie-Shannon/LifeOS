# LifeOS v6.0.0-alpha.2 — Group 28 product phase

## Guarded internal automation

Group 28 adds the first deliberately narrow execution bridge. Approval remains separate from execution. A Low-risk allowlisted internal review-note action can execute only after the global execution gate is explicitly resumed, the proposal is approved, eligibility is recalculated, a final before/after preview is shown, and the user explicitly confirms execution.

The action is reversible. Before and after snapshots, execution identity, timestamps, source references, result state and Undo state are retained. Undo requires an explicit action and restores the exact prior fictional internal state only when the target has not changed incompatibly.

External writes, communication, mailbox, calendar, financial, destructive, script, process, plugin and AI actions remain blocked. There is no background worker, scheduler, timer-triggered execution, startup execution, unattended execution or automatic retry.

Screenshot evidence is intentionally deferred until the product phase is manually verified.
