# LifeOS Desktop v9.0.0-alpha.1 — Group 46 Integration Control Centre

Group 46 creates the permanent system-level control surface and shared contracts for future integrations.

## Added

- Integration Control Centre under Settings without adding a ninth workspace.
- Provider catalogue, provider capability and connected-account models.
- Personal, Work, Business, Family / Household and Other account classification.
- Deterministic connection lifecycle with explicit invalid-transition rejection.
- Required/Optional permission catalogue separated from Granted/Missing/Revoked/Not Requested state.
- Per-capability freshness and health instead of one misleading provider-wide green state.
- Last attempt, last success, last error and next eligible sync state.
- Explicit reconnect, revoke and disconnect review flows.
- Disconnect retention choices that protect accepted LifeOS records.
- Ordered connector audit history.
- Provider-neutral adapter and normalized connector-result contracts.
- Atomic local JSON persistence outside Git.
- Deterministic fictional Microsoft, Google and local proof providers/accounts.

## Corrected before screenshot approval

- Replaced the separate borderless Control Centre window with an embedded Settings subpage inside the existing LifeOS shell.
- Preserved the normal title bar, workspace rail, top bar, resizing and window movement.
- Added **Back to Settings** and Escape navigation.
- Restored the normal workspace header and metrics when leaving the embedded subpage.
- Kept only reconnect, revoke and disconnect confirmations as owned modal dialogs.
- Preserved the established custom ComboBox presentation for Theme, Accent, Density, Startup workspace and Text scale.

## Safety boundary

No live provider credentials, OAuth consent, Microsoft/Google account connection, external data ingestion, Integration Inbox Group 47 logic, background synchronization or external write action is included. The Assistant and Emergency Stop boundaries remain unchanged.

## Pack boundary

Pack 1 implements and validates the product surface but contains no screenshots. Group 46 closes only after Pack 2 commits exactly eight approved evidence screenshots and final validation documentation.
