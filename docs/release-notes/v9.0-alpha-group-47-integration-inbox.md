# LifeOS Desktop v9.0.0-alpha.2 — Group 47 Integration Inbox

Group 47 creates the permanent normalized review pipeline through which future provider records enter LifeOS.

## Added

- Embedded Integration Inbox reachable from the top-bar Review action, Settings / Integrations and the existing Assistant route.
- Normalized candidate contracts for message, calendar event, contact/person, file/document, task, financial item and generic provider record.
- Candidate states: New, Needs Review, Duplicate Suspected, Conflict, Accepted, Rejected, Ignored, Superseded and Source Removed.
- Provider, account, external ID, capability, source timestamp, imported timestamp, freshness and raw-reference provenance.
- Deterministic normalized fingerprints and provider/account/external-ID idempotency keys.
- Idempotent re-import with explicit audit rather than duplicate creation.
- Changed external-record conflict creation and field-level current/candidate comparison.
- Duplicate choices: link existing, keep separate, replace candidate, ignore or reject.
- Accepted-candidate links to authoritative LifeOS records without duplicate editors.
- Preview-first low-risk batch review with cancellation and explicit application.
- Malformed-candidate quarantine.
- Source-removal tombstones that preserve normalized fields and provenance.
- Ordered audit entries for imports, quarantine, duplicate/conflict decisions, acceptance, linking, batches and source removal.
- Deterministic fictional scenarios covering every normalized candidate type.
- Dynamic Integration Inbox review count in the v8 top bar.
