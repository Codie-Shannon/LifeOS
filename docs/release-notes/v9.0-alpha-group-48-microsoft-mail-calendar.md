# LifeOS Desktop v9.0.0-alpha.3 — Group 48 Microsoft Mail and Calendar

Group 48 establishes the single LifeOS Microsoft provider registration boundary and implements the first real Microsoft features: read-only Outlook Mail and Microsoft Calendar.

## Added

- Embedded Microsoft Provider Setup under Settings.
- One provider configuration for the planned Microsoft family.
- Authorization-code flow with PKCE and localhost Desktop redirect.
- DPAPI-protected per-user token storage outside Git.
- Multiple Microsoft accounts with explicit Personal/Work classification.
- Safe Microsoft identity verification.
- Bounded Outlook Mail folder/date/item reads.
- Attachment metadata reads without automatic attachment-body download.
- Bounded Microsoft Calendar date-window/item reads.
- Message and calendar normalization through Integration Inbox.
- Reviewable follow-up, waiting-on, work-context and schedule suggestions.
- Incremental permission catalogue: Mail and Calendar only in Group 48.
- Expired, revoked, offline, throttled and partial-permission recovery states.
- Disconnect that deletes local token material while retaining accepted LifeOS records.
- Deterministic fake-transport tests for Graph parsing and Inbox routing.

## Read-only scope boundary

Requested only when the user connects:

- `User.Read`
- `Mail.Read`
- `Calendars.Read`
- `openid`
- `profile`
- `offline_access`

No external write scope or action exists.

## Later capability boundary

Contacts/People, OneDrive, SharePoint, Teams, Power BI and Power Automate are recorded in the complete provider catalogue but remain Not Requested and unimplemented.
