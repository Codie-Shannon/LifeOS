# LifeOS Desktop v9.0.0-alpha.4 — Group 49 Microsoft Files

Group 49 extends the existing Microsoft provider registration with incremental delegated read-only OneDrive and SharePoint metadata capability.

Implemented boundaries:

- reuse of the Group 48 Microsoft identity, local token boundary and Integration Inbox;
- incremental `Files.Read` and `Sites.Read.All` consent catalogue with no write scopes;
- explicit selected-drive, selected-folder, selected-site and selected-library boundaries;
- normalized file/folder metadata and source provenance;
- metadata-first synchronization with no automatic body download;
- idempotent re-import and visible source-removed/permission-lost states;
- partial-source failure reporting that fails closed;
- accepted candidate linking through the existing review-first inbox.

No upload, rename, move, delete, sharing, permission mutation or external write action is introduced.
