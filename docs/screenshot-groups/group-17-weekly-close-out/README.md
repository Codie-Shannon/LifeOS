# Group 17 - Weekly Close-Out

LifeOS Desktop v4.7 completes the weekly operating loop across work, money, waiting-on, proof, receipt evidence, and next-week pressure.

## What this group proves

- Completed and evidence-backed items can be closed deliberately.
- Unfinished work can be rolled into next week with an owner, next action, and reason.
- Waiting and blocked work remains visible without consuming active sprint time.
- Expected money remains excluded from safe money until paid and cleared.
- Proof and evidence gaps remain visible.
- Receipt/OCR candidates remain source-gated and review-first.
- Work Pipeline items feed the weekly close-out loop.
- The original manual close-out form and history remain available.

v4.7 is local/manual. It does not close real projects, send follow-ups, create invoices, move money, reconcile accounts, trust OCR automatically, sync external systems, or run AI actions.
