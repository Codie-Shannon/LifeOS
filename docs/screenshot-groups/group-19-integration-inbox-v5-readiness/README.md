# Group 19 - Integration Inbox + v5 Readiness

## Release

LifeOS Desktop v4.9.

## Purpose

Group 19 proves the final v4 spine layer: external-looking data enters LifeOS as read-only previews with provenance, duplicate checks, review states, and explicit manual handoff contracts.

## Verified behavior

- External records are staged as read-only previews.
- Source provenance and evidence remain attached.
- Duplicate suspicion blocks clean acceptance.
- Accept, defer, reject, and link are explicit review actions.
- Accepted previews still require a separate target-module handoff.
- Imported and expected money remains excluded from safe money.
- No live connector or automatic external mutation is active.
- The linked-handoff screenshot proves the final review transition to `Linked` and `Trusted`.

## Boundary

v4.9 does not activate Gmail, Outlook, Calendar, Xero, SharePoint, Drive, OCR, banking, OAuth, live polling, automatic item creation, automatic updates, or AI actions.

## Next lane

v5.0 can begin with one narrow read-only connector using strict scopes, preview review, duplicate detection, provenance, and reversible manual handoff.
